For our creative section of the server, Plots, we have quite a few commands. Here we'll explain most of those, what they do, and how to use them properly.

This is meant to be an exhaustive list, it will contain all if not a majority of commands

Basics

Using the command: "/plots help" will show all possible help commands for the plots plugin. 
One important information for these commands though, remember to type in "/plots" not just "/plot". "/plot" will not work, unless with the "s" at the end.

To confirm any plot action, use the "/plots confirm" command.

Info commands

"/plots info", using this command will show you the details for the current plot you're standing / flying on. It will show informations such as:
Plot ID, Creation Date, Owner, Biome and more.

"/plots list", using this command will give you options for sub-commands such as:
forsale, mine, shared, world, all, unowned. The correct usage would be: "/plots list [sub-command]"
Example: /plots list mine
It will show all your owned plots.

Teleport commands

"/plots home", this command has two ways to work.
Firstly, you can simply type "/plots home", but in the case that you've got multiple plots, you can type "/plots home [number of plot;number of plot]
Example: /plots home 1;0. You can see the number of your plot using "/plots info".

"/plots visit", the correct usage of this command is "/plots visit [player name]"
Example: /plots visit Paopou__

"/plots middle" using this command you can teleport yourself into the middle of the current plot you're standing on.

Claiming commands

"/plots claim" let's you claim a specific plot you're standing on, aslong as it is unclaimed.
"/plots claim auto" will let you claim the nearest plot possible.

Keep in mind, you are only able to own one plot.



Settings commands 

 which may be added sometime in the near future.

"/plots setowner [player name]" this command sets a certain player the owner of the plot you're on, although this command is not available to the average player, yet.

"/plots add [player name]" using this command you are able to add players to your plot which directly gives them build access as long as the plot owner is online.

"/plots trust [player name]" this command simply does the same as the add command but, it gives the trusted player World Edit permissions as well as build permissions while the plot owner is not online.

"/plots remove [player name]" this command removes people from your trusted list, whitelisted (add) list or just your black list.

"/plots deny [player name]" using this command you are able to blacklist people from your plot.

"/plots kick [player name]" this command gives you the ability to kick people from your plot.

"/plots merge" lets you merge multiple plots into one (side note, not possible atm.)

"/plots unlink" lets you un-merge multiple plots into one (as the merge is not possible, there is no use for unlink as of this time.)

"/plots sethome" lets you set a new home point in your plot.

"/plots alias (set | remove) {alias}" lets you set a name for your plot.

"/plots desc {description}" lets you add a description for your plot, although this is not accessible as for now.

"/plots music" lets you set a music(disc) for your plot.

"/plots biome" lets you change the biome of your plot.
Example: /plots (set)biome badlands

"/plots flag" opens a help menu for the flag command, helping you further in terms of flags. (No, not the flags of countries etc.)

"/plots done" marks your plot as finished.

"/plots continue" marks your plot as unfinished again and lets you build.

"/plots set" gives you an option of things which you are able to do on your plot, such as:
set biome, set alias, set home, set floor etc. etc.

"/plots clear" lets you clear your plot (Warning, everything you've built WILL be removed.

"/plots delete" lets you delete your entire plot, unclaiming it for you. (Warning, everything you've built WILL be removed as well as unclaimed.)

Things will be updated/edited after changes have been made, more commands are accessible or commands are restricted.
