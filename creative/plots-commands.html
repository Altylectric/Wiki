The Creative Mode section of our server uses a plugin known as Plots. Plots has quite a few commands. This section explains the commands, what they do, and how to utilise them properly and to their full extent.

This is meant to be an exhaustive list; it will contain all, if not the majority of the commands. Further, all commas and periods in quotations are in place for proper punctuation and are NOT to be used as part of the command. Thank you for your understanding.

Basic Commands

The command: "/plots help" will show all possible help commands for the plots plugin. 
One important piece of information about these commands, though, is to remember to type in "/plots," not just "/plot." "/plot" will not work unless it has the "s" at the end.

The command: "/plots confirm" will be used to confirm any plot action. This allows the system to know you intentionally typed and submitted your previous command.

Informational Commands

The command: “/plots info" will show you the details for the current plot you're standing/flying on. It will show information such as:
Plot ID, Creation Date, Owner, Biome, and more.

The command: "/plots list" will give you potential options as sub-commands, such as:
For sale, mine, shared, world, all, unowned. The correct usage would be: "/plots list [insert sub-command here]."
For example, the command “/plots list mine,” as mentioned above, will display all your owned plots to you in a small menu.

Teleportation Commands

The command: "/plots home" will set your home based on player input. Your home is your designated main area for teleportation.
Typically, when starting, you can type “/plots home,” but in the unlikely case that you've got multiple plots (as plots are currently restricted to one per player), you can type “/plots home [insert number of plot here; insert number of plot here]”
For example, “/plots home 1;0.” You can view plot numbers using the command “/plots info.”

The command: “/plots visit” will teleport you to the home plot of the player mentioned in the command. The correct usage of this command would be "/plots visit [insert player name here]."
For example, to teleport to the home of user Paopou__ or Viceroy_la_Richt, you would submit the command "/plots visit Paopou__" or “/plots visit Viceroy_la_Richt.”

The command: "/plots middle" will teleport you to the exact centre of the current plot you're within.

Land-Claim Commands

The command: "/plots claim" will allow you to claim a specific plot you're standing on as long as the requirement of it being unclaimed is fulfilled.

The command: "/plots claim auto" will allow you to claim the nearest unclaimed plot (within a reasonable distance).

Please keep in mind that players are currently restricted to the ownership of one plot.

Settings Commands 

The following commands are debug/administrative commands and are NOT currently available to non-team member players. Thank you for your understanding and cooperation.

The command: "/plots setowner [insert player name here]" will set a certain player as the owner of the plot you're on, although this command is not available to the average player yet.

The command: "/plots add [insert player name here]" will add the specified player to your plot, which directly gives them build access as long as the you are online.

The command: "/plots trust [insert player name here]" will achieve the same goal as the “/plots add” command, but it gives the specified player World Edit and Build permissions while the you are offline.

The command "/plots remove [insert player name here]" will remove the specified player from your trusted list, whitelist, or blacklist.

The command: "/plots deny [insert player name here]" will blacklist the specified player from your plot.

The command: "/plots kick [insert player name here]" this command gives you the ability to kick people from your plot.

The command: "/plots merge" lets you merge multiple plots into one.

The command: "/plots unlink" lets you un-merge multiple plots into one (as the merge is not possible, there is no use for unlink as of this time.)

The command: "/plots sethome" lets you set a new home point in your plot.

The command: "/plots alias (set | remove) {alias}" lets you set a name for your plot.

The command: "/plots desc {description}" lets you add a description for your plot, although this is not accessible as for now.

The command: "/plots music" lets you set a music(disc) for your plot.

The command: "/plots biome" lets you change the biome of your plot.
Example: /plots (set)biome badlands

The command: "/plots flag" opens a help menu for the flag command, helping you further in terms of flags. (No, not the flags of countries etc.)

The command: "/plots done" marks your plot as finished.

The command: "/plots continue" marks your plot as unfinished again and lets you build.

The command: "/plots set" gives you an option of things which you are able to do on your plot, such as:
set biome, set alias, set home, set floor etc. etc.

The command: "/plots clear" lets you clear your plot (Warning, everything you've built WILL be removed.

The command: "/plots delete" lets you delete your entire plot, unclaiming it for you. (Warning, everything you've built WILL be removed as well as unclaimed.)

Things will be updated/edited after changes have been made, more commands are accessible or commands are restricted.
